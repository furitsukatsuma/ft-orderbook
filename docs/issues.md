# Issue の書き方（おさらい）

**「教えてね」「こうしたい」は Issue で OK** です。  
コードを直した人は **Pull Request** を使います。

## Issue とは

GitHub の **やること・困りごと・要望のチケット** です。  
このリポジトリ専用の掲示板のようなもの。

## こんなとき Issue に書いてね

| 内容 | 例 |
|------|-----|
| 使いにくい | 「注文の流れがわかりにくい」 |
| 見えない・UI | 「背景が暗くて読みにくい。色を変えたい」 |
| バグ | 「板が更新されない」 |
| 要望 | 「〇〇機能が欲しい」 |
| 質問 | 「MCP の設定はどこ？」 |

## 書き方の例

**タイトル:** `UI: 売り板の文字色をもっと明るくしたい`

**本文:**

```text
orderbook UI の色が暗くて見づらいです。
売り板・買い板のコントラストを上げられると助かります。

環境: Chrome / macOS
```

## Issue と PR の使い分け

| | Issue | PR |
|---|--------|-----|
| 向き | 意見・要望・バグ報告・質問 | コード変更の提案 |
| 例 | 「色変えたい」 | 「色を直したパッチ」 |

```
要望を Issue で書く
    ↓（直せる人がいれば）
fork → 修正 → PR
    ↓
マージ → Issue を Close
```

## 返信について

- メンテナーが **毎回コメントを返すとは限りません**
- 初回は **絵文字リアクション**（❤️ 🎉 🚀 など）が自動で付くことがあります
- 急ぎでなければ **DM は不要** です

## 作成手順

1. https://github.com/furitsukatsuma/ft-orderbook/issues
2. **New issue**
3. タイトル + 本文を書いて **Submit**

協力してコードまで直す場合は [CONTRIBUTING.md](../CONTRIBUTING.md) も読んでください。
